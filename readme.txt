База данных аналога сайта Geekbrains
Хранит информацию о студентах, преподавателях, программах обучения,
курсах и расписаниях уроков.

Для заполнения базы данными был использован ресурс http://filldb.info.
Для проверки согласованности данных использовались процедуры и триггеры с обычной(медленой) вставкой,
чтобы отследить невалидные данные.
Если данные проверять не нужно - пакетная вставка.

Для решения задач базы данных были созданы представления:
1) student_info: показывает следующую информацию о студенте: имя-фамилия, его группу, программу обучения, специализацию, все его курсы, всех его преподавателей и кол-во студентов в группе.
2) students_debt_info: таблица всех долгов студента: id студента, его имя-фамилия, id курса, номер урока, ДАТА СТАРТА КУРСА, id преподавателя.
3) courses_by_group - все курсы группы;
4) teacher_student_practical;

Процедуры:
students_fill_sp, teachers_fill_sp - проверка одновременной вставки в две таблицы

Триггеры:
1) edu_groups_check_before_insert - проверяет согласованность данных при создании новой группы (
чтобы специализация соответствовала программе);
2) messages_check_before_insert - проверяет новое сообщение (не себе, не дубликат)
3) schedule_check_before_insert - проверяет ведет ли препод такой курс,
есть ли у группы такой курс, есть ли в курсе урок с таким номером, время уроков с 10 до 20,
уроки по порядку для каждой группы(номер урока курса для группы), у одной группы или препода
нет одновременно уроков(разница в 5 часов), следующий урок у студента может начаться не раньше
чем через 48 часов, знает ли наставник этот курс;
4) students_practicals_check_before_insert - проверяет что у студента есть урок, которому
соответствует пз,  задание сдается не позднее чем через 2 недели после начала урока и не
раньше конца урока;
5) teachers_practicals_check_before_insert - проверяет что задание выполнено и студент учится
у этого преподавателя;
6) job_checked_after_insert - преподаватель посылает студенту сообщение о том, что проверил
практическое задание;
